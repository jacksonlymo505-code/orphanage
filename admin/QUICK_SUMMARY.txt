ROOT CAUSE:
═══════════════════════════════════════════════════════════════════════════
The donors table had:
  status: ENUM('active', 'inactive')
  
But code tried to save:
  status: 'approved' OR 'rejected'

MySQL rejected the invalid enum value, leaving status field EMPTY!


THE FIX:
═══════════════════════════════════════════════════════════════════════════
1. Added new column: approval_status ENUM('pending', 'approved', 'rejected')

2. Updated code in admin/manage_donors.php to use:
   - approval_status = 'approved' (tracks admin decision)
   - status = 'active' (tracks if donor can login)
   - status = 'inactive' (if rejected)

3. Verified SMS sends to phone number with email and password
